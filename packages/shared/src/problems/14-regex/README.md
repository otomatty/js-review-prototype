### S3 学習ポイント

- 正規表現リテラル `/pattern/flags` の構文。 `/\d/` で 1 文字、 `/\d+/` で 1 文字以上の連続
- フラグ `g` 無しは「最初の 1 件」、 `g` 付きは「すべて」
- アンカー `^` `$` で「文字列全体」 にマッチを限定。 単独のパターンは部分一致になる
- `[^@\s]+` のような文字クラスで「除外したい文字」 を指定する

## S4 で扱う問題 (5 問)

| # | ID | 主題 |
|---|---|---|
| 01 | `S4-Ch14-01-parse-date` | キャプチャグループ `(...)` で `YYYY-MM-DD` を分解 |
| 02 | `S4-Ch14-02-is-postal-code` | 量指定子 `{n}` で桁数固定の入力検証 |
| 03 | `S4-Ch14-03-snake-to-camel` | `replace` + コールバック関数で整形 |
| 04 | `S4-Ch14-04-extract-hashtags` | `matchAll` + キャプチャでトークン抽出 |
| 05 | `S4-Ch14-05-parse-query-string-capstone` | [卒業課題] `matchAll` + 2 キャプチャ + `reduce` でオブジェクト化 |

### S4 学習ポイント

- **キャプチャグループ** `(...)` で「マッチした部分文字列」 を後から取り出せる。 `match` の戻り値は
  `[0]` が全体、 `[1]` `[2]` ... がキャプチャに対応する
- **量指定子** `{n}` (ちょうど n 回)、 `{n,m}` (n〜m 回) で **桁数を厳密に制御** する。 入力検証で必須
- `replace(re, callback)` の **コールバック関数** は `(マッチ全体, キャプチャ1, ...)` を受け取り、 戻り値が置換文字列になる。
  整形 (大文字化・小文字化・並び替え) に使う
- `matchAll(re)` は **イテレータ** を返す。 `[...s.matchAll(re)]` で配列化してから `map` / `reduce` に流し込むのが定番
