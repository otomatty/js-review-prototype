> **🆕 この章は S4 で初めて出ます。** S0-S3 では登場しない「アルゴリズムを自分で組み立てる」 (探索・整列・再帰) を S4 から扱います。

| # | タイトル | 主に学ぶ概念 |
|---|---|---|
| 1 | 線形探索で値の添字を返す | `for` で先頭から走査する素朴な探索 (`O(N)`) |
| 2 | 二分探索でソート済み配列を探す | ソート済み前提で範囲を半分にする (`O(log N)`) |
| 3 | `Array.sort` の比較関数で文字列を長さ順に並べる | `(a, b) => a.length - b.length` の比較ルール |

## S4 で身につけること

- **計算量への意識付け**: 線形探索 (`O(N)`) と二分探索 (`O(log N)`) を並べて比べ、 「ソート済みかどうか」 が探索コストを変えることを実感する。
- **ソートの中身**: 組込み `.sort` を呼ぶだけでなく、 バブルソートを自分で書いて `O(N^2)` の挙動を理解する (なぜ大きな配列で組込みより遅いのか)。
- **再帰の入口**: ベースケースを書き忘れると無限再帰になることや、 素朴なフィボナッチが `O(2^N)` で爆発することを体感し、 後の章のメモ化・DP へつなげる。
- **DFS の入口**: 卒業課題のネスト配列合計で、 「データ構造そのものを再帰で辿る」 感覚を掴む。

## S5 問題ラインナップ

| # | タイトル | 主に学ぶ概念 |
|---|---|---|
| 1 | メモ化でフィボナッチを高速化する (トップダウン DP) | `Map` をキャッシュに使う再帰 — `O(2^N)` → `O(N)` |
| 2 | 階段の登り方を DP テーブルで数える (ボトムアップ DP) | 漸化式を配列で順に埋める動的計画法の典型形 |
| 3 | [卒業課題] 二分探索でソート済み配列の範囲内要素数を数える | lower_bound への発展と 2 回の二分探索による範囲集計 (`O(log N)`) |

## S5 で身につけること

- **「同じ計算を 2 度しない」 設計**: S4 の素朴な再帰 fib (`O(2^N)`) を、 `Map` キャッシュで `O(N)` に落とす感覚。 メモ化 (トップダウン DP) は再帰の自然な延長として書ける。
- **ボトムアップ DP の典型形**: 漸化式を 「小さい i から順に DP テーブルを埋める」 形に落とす練習。 階段の登り方は fib と同じ漸化式だが、 ベースケースの取り方が違うところに注目する。
- **二分探索の応用 (境界探索)**: 「等しい値を 1 つ探す」 から 「ある条件を満たす最小の添字を探す (lower_bound)」 への発展。 これは paiza A 帯以降や競プロでも頻出のテクニック。
- **計算量に基づく設計判断**: 卒業課題で `filter` (O(N)) と二分探索ベース (O(log N)) を比較し、 「同じ集計問題でも、 ソート済みかどうかで使うべき道具が変わる」 ことを体感する。

### 計算量比較サマリ (S4 → S5 の道のり)

| 問題 | 素朴な実装 | S5 での改善 |
|---|---|---|
| フィボナッチ数 | 再帰 (`O(2^N)`) | メモ化再帰 / DP テーブル (`O(N)`) |
| 階段の登り方 | 再帰 (`O(2^N)`) | DP テーブル (`O(N)`) |
| 範囲内要素数の集計 | `filter` 線形走査 (`O(N)`) | 二分探索 × 2 (`O(log N)`) |
