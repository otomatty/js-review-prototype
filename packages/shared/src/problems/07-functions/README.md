### S2 学習ポイント

- `function 名前() { ... }` で宣言。 宣言しただけでは実行されない。 `名前()` で呼び出して初めて中身が動く
- `return` で値を返す。 書かないと暗黙的に `undefined` が返る
- アロー関数 `() => 値` は暗黙 return、 `() => { return 値; }` は明示 return
- デフォルト引数 `f(x = 既定値)` で省略可能にできる
- 早期 return で if/else のネストを浅く保つ
- 関数を **小さな部品** として分割すれば、 後から組み合わせて複雑な処理を作れる

## S4 で扱う問題 (5 問、 5 番目が卒業課題)

| # | ID | 主題 |
|---|---|---|
| 01 | `S4-Ch07-01-once` | 1 度だけ実行する高階関数 (`once`) |
| 02 | `S4-Ch07-02-memoize` | Map で結果をキャッシュする高階関数 (`memoize`) |
| 03 | `S4-Ch07-03-pipe` | 残余引数で受けた関数を左から順に適用 (`pipe`) |

### S4 学習ポイント

- **高階関数の自作**: `once` / `memoize` / `curry` のような汎用ヘルパを 「自分の手で」 書けるようになる
- **クロージャでの状態保持**: 「呼ばれたか」 や Map をクロージャに閉じ込めて、 返した関数に状態を持たせる
- **可変長合成**: 残余引数 `(...fns)` で関数列を受け、 `reduce` で順番に適用する
- **関数の分割と合成**: 大きな処理を 「**小さな 1 引数関数**」 と 「**それを束ねる高階関数**」 に分けて構成する (卒業課題のテーマ)
- **早期 return の活用**: 検証ループのように 「最初に見つかったものを即座に返す」 形を組み立てる
