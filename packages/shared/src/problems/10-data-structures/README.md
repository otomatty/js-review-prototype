### S5 学習ポイント

- **「データ構造を選ぶ」 のが本題**: カウンタは Map、 重複除去は Set、 木は再帰、 辺の配列は隣接リスト (Map + Set)、 という対応関係を体で覚える。 Object でも書ける問題でも、 「動的な文字列キー」 「挿入順保持」 「重複判定 O(1)」 が要るときは **意識的に Map / Set を選ぶ** のが S5 の作法
- **Map と Set の役割分担**: 1 つの問題の中で *カウンタ Map* と *重複除去 Set* を同時に使う (`01-tag-cooccurrence`)、 *隣接リスト Map* と *メンバー Set* と *訪問済み Set* と *距離 Map* を同時に使う (`03-friends-within-hops-capstone`) ように、 同じ問題でも用途別に複数の Map / Set を立てて整理する
- **再帰でツリー走査**: ヘルパー関数 `function visit(node)` を内側に定義し、 「自分の貢献 + 子の `visit` 結果の合計」 を return する形が定型 (`02-category-tree-aggregate`)。 ツリーの深さに依存せずに自然に書ける
- **グラフは「辺の配列」 のまま走査しない**: グラフ問題が出たらまず **隣接リストに変換する前処理** を入れるのが王道。 BFS は `visited` Set + キュー (配列の `push` / `shift`) + `while` ループ + `if (hop >= maxHops) continue;` の打ち切りで定型化できる
- **集計の整形パイプライン**: S4 で学んだ 「`Map → [...entries()] → map → sort → slice`」 を S5 でも引き続き使う。 ただし S5 では並べ替えの 2 段ソート (主キー数値降順 + 副キー文字列昇順) や、 上限件数 `topN` の早期 return / `Math.max(0, topN)` を組み合わせて 「設計として穴がない」 関数に仕立てる
