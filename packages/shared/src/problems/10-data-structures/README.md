# Ch10. データ構造

`Map` / `Set`、 集計、 探索に使うデータの持ち方。 「配列・オブジェクトでも書けるが、 専用のデータ構造を使うとどう変わるか」 を体感する。 再帰の入り口もここで触れる。

> **🆕 この章は S4 で初めて出ます。** S0-S3 では登場しない「キー付きコレクション」 (`Map` / `Set`) を S4 から扱います。

## 含まれるステージ

- **S4** アルゴリズム — `Map` / `Set` での重複排除・グルーピング・カウント (実装済み)。
- **S5** 設計演習 — 探索問題や再帰問題で適切なデータ構造を選ぶ (Phase 9 後半で追加予定)。

## S4 問題ラインナップ

| # | タイトル | 主に学ぶ概念 |
|---|---|---|
| 1 | Set で配列から重複を取り除く | `new Set(arr)` の基礎 |
| 2 | Map で各要素の出現回数を数える | `Map` をカウンタとして使う |
| 3 | Map で文字列を先頭文字でグルーピング | 「未登録なら空配列で初期化」 パターン |
| 4 | Set で 2 つの配列の共通要素を求める | `Set.has` の O(1) を活用 |
| 5 | Map で最頻出の要素を見つける | 集計と「現在の最大」 更新を 1 周で |
| 6 | Map で最初の「1 回しか出ない要素」 を探す | 前計算 → 2 周目で順序を保って探索 |
| 7 | [卒業課題] 売上を Map で集計し上位 N 件を返す | Map 集計 → `entries()` → `sort` → `slice` |
